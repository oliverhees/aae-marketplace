## Task-Zuweisung Guidelines

### Wer macht was?

| Task-Typ | Zugewiesen an |
|----------|---------------|
| User Research, Wireframes | UX Designer |
| Visual Design, Components | UI Designer |
| Microcopy, Texte | Copywriter |
| Pages, Layouts, Client State | Frontend Dev |
| API Routes, Business Logic | Backend Dev |
| Database Schema, RLS | Supabase Specialist |
| Auth, Security | Auth Expert |
| E2E Tests | QA Lead |
| Code Review | Code Reviewer |
| CI/CD, Deployment | DevOps Engineer |

### Task-Format

Beim Anlegen eines Tasks:

```markdown
| [Klare Beschreibung was zu tun ist] | [Agent] | pending | [CRITICAL/HIGH/MEDIUM/LOW] | - |
```

Beispiel:
```markdown
| Dashboard Layout mit Sidebar implementieren | Frontend Dev | pending | HIGH | Wartet auf Wireframes |
```
